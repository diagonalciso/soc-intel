Introduction

SOCint is a unified threat intelligence platform (TIP) built around STIX 2.1 standard. It ingests 26+ external threat feeds, stores intelligence as standardized objects, enriches data with external APIs, and provides analysts with tools for threat hunting, case management, detection rules, and compliance reporting.

Key Features:

  • 26 Built-in Connectors — OTX, MISP, CISA KEV, NVD, Ransomware.live, dark web feeds, and more
  • STIX 2.1 Standards — All threat intel normalized to STIX 2.1 format with custom extensions
  • Threat Hunting — Pivot searches correlating IOCs, threat actors, malware families, and detection rules
  • Case Management — Track investigations with cases, observables, tasks, and team collaboration
  • Detection Rules — Create and manage Sigma rules, YARA rules, and custom detection logic
  • Dark Web Monitoring — Automated scanning of dark web markets and forums for mentions
  • Enrichment — Enrich IOCs with GreyNoise, VirusTotal, OTX, AbuseIPDB data
  • Compliance — NIST 800-53 mapping, CSF tags, and audit trails for regulatory requirements
  • Real-time Alerts — SSE stream for live alert notifications on new IOCs and matches

Getting Started

1. Login & Registration

First-time user:
  1. Navigate to SOCint web interface (default: http://localhost:3000)
  2. Click "Register" on the login page
  3. Enter email, username, password, and optional organization name
  4. Click "Create Account"
  5. You'll be logged in and redirected to the dashboard
Existing user:
  1. Enter email and password on login page
  2. Click "Login"
  3. You're authenticated with a JWT token valid for 24 hours
Admin note: The first user to register becomes an admin automatically. Subsequent users are analysts by default. Admins can change roles via user management.

2. API Keys

For programmatic access, create API keys:

  1. Click your profile icon (top right) → "Settings"
  2. Select "API Keys"
  3. Click "Create New Key" and give it a label (e.g., "integration-bot")
  4. Copy the key immediately — it won't be shown again
  5. Use it in API requests: Authorization: Bearer YOUR_API_KEY
Security: Treat API keys like passwords. Never commit them to version control. Rotate regularly. Revoke keys you no longer use.

Working with Intelligence

Searching STIX Objects

Basic Search:

  1. Click "Intelligence" in the main menu
  2. Use the search bar to find IOCs, malware, threat actors, etc.
  3. Results return all STIX object types: ipv4-addr, domain-name, file, threat-actor, malware, attack-pattern, etc.

Advanced Filtering:

  • By Type: Filter by object type (IOC, malware, actor, etc.)
  • By TLP: Filter by Traffic Light Protocol marking (CLEAR, GREEN, AMBER, RED)
  • By Source: Filter by connector name (OTX, MISP, CISA, etc.)
  • By Confidence: Filter by confidence score (0–100)
  • By Sightings: Find indicators with N or more sightings
TLP Meanings:
  • TLP:CLEAR — Unlimited sharing
  • TLP:GREEN — Community sharing with handling restrictions
  • TLP:AMBER — Limited to need-to-know recipients
  • TLP:RED — Not for sharing outside recipient's organization

Viewing Object Details

Click any object to see full details:

  • Core attributes — value, type, first/last seen, confidence
  • Relationships — linked objects (e.g., IP used by malware, malware used by threat actor)
  • Sightings — sources and dates this indicator was observed
  • Enrichment — external API results (GreyNoise, VirusTotal, etc.)
  • TLP & Tags — traffic light protocol and custom tags
  • History — who created/modified object and when

Creating Custom Intelligence Objects

Analysts can manually add intelligence:

  1. Click "+ Create Object" in the Intelligence section
  2. Select object type:
    • ipv4-addr / ipv6-addr — IP addresses
    • domain-name — Domain names
    • file — File hashes (MD5, SHA1, SHA256)
    • url — URLs
    • threat-actor — Threat groups
    • malware — Malware families
    • attack-pattern — MITRE ATT&CK techniques
    • relationship — Links between objects
  3. Fill in object attributes (value, names, descriptions, etc.)
  4. Set TLP level (default: AMBER)
  5. Add tags for tracking
  6. Click "Create"

Editing & Deleting Objects

  • Edit: Click the pencil icon on an object → modify fields → "Save"
  • Delete: Click the trash icon → confirm deletion (admins only)
  • Relationships: Create links between objects (e.g., "IP hosted malware X")

Threat Hunting

Pivot Search

SOCint's most powerful feature for correlation hunting:

  1. Click "Hunting" in the main menu
  2. Enter a search term in the pivot box:
    • IOC (IP, domain, hash, URL)
    • Threat actor name (e.g., "APT1", "Lazarus")
    • Malware family (e.g., "Emotet", "Trickbot")
    • MITRE technique ID (e.g., "T1566")
  3. Results return:
    • STIX Objects — Matching IOCs, actors, malware
    • Sigma Rules — Detection rules covering techniques used by results
    • YARA Rules — File-based detection rules
    • Threat Actors — Groups linked to your search term
    • Malware Families — Malware associated with actors/techniques
  4. Click any result to drill down for investigation

Example hunting workflow:

  1. Incident team reports IP 192.0.2.10 in firewall logs
  2. Pivot search on the IP → returns 3 matching IOCs
  3. View relationships → IP is used by malware "ZeuS"
  4. Search malware → links to threat actor "Carbanak"
  5. View techniques used by Carbanak → T1566 (phishing), T1547 (persistence)
  6. Fetch Sigma rules for T1566 → deploy to SIEM
  7. Create case "Carbanak Investigation" and link observables

Case Management

Creating a Case

  1. Click "Cases" in the main menu
  2. Click "+ New Case"
  3. Fill in:
    • Title — Brief case name (e.g., "Emotet Campaign Jan 2024")
    • Description — Detailed context
    • Severity — low, medium, high, critical
    • TLP — Traffic Light Protocol level
    • Tags — Custom tags for filtering (e.g., "ransomware", "apt1")
  4. Click "Create Case"

Working with Observables

Observables are artifacts linked to a case (IOCs, file hashes, URLs, usernames, etc.):

Add Observable:
  1. Open a case → click "+ Add Observable"
  2. Select type: ipv4, domain, file hash, url, username, email, etc.
  3. Enter value
  4. Toggle "Is IOC" if it's an indicator of compromise
  5. Add tags
  6. Click "Add"

Bulk Add Observables: Paste comma-separated or newline-separated IOCs in bulk import.

Tasks & Workflow

Track investigation progress with tasks:

  1. Click "+ Add Task" within a case
  2. Enter task title (e.g., "Check if IP in passive DNS")
  3. Add optional description
  4. Assign to team member or mark unassigned
  5. Mark complete when done

Comments & Collaboration

  • Add Comment: Click comment icon → add note → "Post"
  • Mention Team: Type @username to notify colleagues
  • Edit/Delete: Own comments can be edited or deleted

Case Status & Lifecycle

Cases progress through statuses:

  • Open — Active investigation
  • In Progress — Currently being worked
  • Closed — Investigation concluded
  • Archived — Old case, hidden by default

Update status via the case detail dropdown.

Compliance Tags

Link cases to compliance frameworks for reporting:

  • NIST 800-53 — Map controls (e.g., SI-4 Information System Monitoring)
  • CSF Tags — NIST Cybersecurity Framework functions (Identify, Protect, Detect, Respond, Recover)

Select tags when creating/editing case → use in compliance reports.

Exporting Cases

  • Click "Export" button → choose format (JSON, CSV, PDF)
  • Exported file includes all observables, tasks, comments, and metadata

Detection Rules

Viewing Rules

Click "Rules" in the main menu to browse detection rules:

Rule Type Purpose Detects
Sigma Rules SIEM-agnostic detection logic Log patterns, behavioral anomalies
YARA Rules File-based detection (malware scanning) Malware strings, PE sections, byte patterns
Custom Rules Organization-specific detection Any IOC or pattern

Creating a Detection Rule

  1. Click "+ Create Rule"
  2. Choose rule type (Sigma, YARA, or custom)
  3. For Sigma rules:
    • Name, description
    • Map MITRE ATT&CK techniques
    • Set severity (informational, low, medium, high, critical)
    • Write YAML-formatted Sigma detection logic
    • Example: detect PowerShell execution with suspicious flags
  4. For YARA rules:
    • Name, description
    • Write YARA detection strings
    • Specify file patterns to scan
  5. Click "Create"

Testing Rules

  • Click "Test" on a rule → upload test file or log sample
  • View matches instantly
  • Iterate until rule triggers on malicious samples only

Deploying Rules

  • Sigma: Export and deploy to Splunk, ELK, Graylog, etc. via SIEM connectors
  • YARA: Export and deploy to malware scanning workflows (yara CLI, integration tools)
  • Status: Mark rule as "Active" or "Inactive" in SOCint

Dark Web Monitoring

Accessing Dark Web Intel

SOCint monitors dark web markets and forums automatically:

  1. Click "Dark Web" in the main menu
  2. Browse feeds by source:
    • Dark web marketplaces (stolen data, credentials)
    • Hacking forums (exploit announcements, threat actor chatter)
    • Paste sites (leaked databases)
  3. Search for:
    • Your organization name or domain
    • Known threat actor handles
    • Specific vulnerabilities or exploits
    • Employee email addresses or names

Alerts on New Mentions

  • Enable "Watch" on your organization or keywords
  • Receive real-time SSE alerts when new mentions appear
  • Create cases directly from dark web findings

Enrichment

On-Demand Enrichment

Enrich IOCs with external intelligence sources:

  1. Search or view an IOC in Intelligence section
  2. Click "Enrich" button
  3. SOCint queries:
    • GreyNoise — Is this IP malicious or harmless Internet noise?
    • VirusTotal — File hash detections, URL analysis
    • OTX — Pulse data, references
    • AbuseIPDB — Abuse reports on IP
  4. Results cached for 24 hours
  5. View enrichment in object details

Bulk Enrichment

  • Select multiple IOCs → click "Enrich Selected"
  • Queued asynchronously, results appear as they complete

Compliance & Reporting

NIST 800-53 Mapping

Link investigation findings to security controls:

  • SI-4: Information System Monitoring — detection of malicious activity
  • CA-8: Penetration Testing — hunting results
  • IR-4: Incident Handling — case management

Tag cases with relevant controls; use in compliance reports.

CSF Mapping

Align findings to NIST Cybersecurity Framework:

  • Identify — Asset/vulnerability discovery
  • Protect — Detection rules deployed
  • Detect — Alerts, hunts, IoC matches
  • Respond — Case management, incident response
  • Recover — Forensics, post-incident tuning

Generating Reports

  1. Click "Reports" in the main menu
  2. Select report type:
    • Threat Summary — Top IOCs, threat actors, techniques by period
    • Compliance Report — Cases mapped to NIST 800-53, CSF
    • Hunting Results — Summary of detection activity
    • Custom Report — User-defined scope and filters
  3. Choose date range, confidence thresholds, TLP levels
  4. Generate → view online or download as PDF/CSV

API & Integration

REST API Overview

SOCint exposes full REST API for programmatic access:

Endpoint Method Purpose
/api/auth/login POST Authenticate and get JWT token
/api/intel/objects GET, POST, PATCH, DELETE STIX object CRUD
/api/intel/objects/search GET Search IOCs with filters
/api/hunting/pivot GET Pivot search across all intel
/api/cases GET, POST Case CRUD
/api/rules GET, POST Detection rule management
/api/enrichment POST Enrich IOCs
/stream GET (SSE) Real-time alert stream

Authentication

curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"secret"}'

# Response:
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}

# Use token in subsequent requests:
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  http://localhost:8000/api/intel/objects?type=ipv4-addr

Example: Search IOCs

curl "http://localhost:8000/api/intel/objects?type=ipv4-addr&q=192.0&tlp=TLP:CLEAR" \
  -H "Authorization: Bearer YOUR_TOKEN"

# Response:
{
  "results": [
    {
      "id": "ipv4-addr--12345678-1234-1234-1234-123456789012",
      "type": "ipv4-addr",
      "value": "192.0.2.10",
      "created": "2024-01-15T10:30:00Z",
      "confidence": 95,
      "tlp": "TLP:CLEAR",
      "source": "OTX",
      "sightings": 15
    }
  ],
  "total": 1
}

Real-Time Alert Stream (SSE)

curl -N -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8000/stream

# Stream will continuously send events:
data: {"type":"new_ioc","ioc":"192.0.2.100","source":"NVD"}
data: {"type":"rule_match","rule":"Emotet C2 Detection","severity":"high"}

FAQ & Troubleshooting

Q: What's the difference between TLP levels?

A: Traffic Light Protocol (TLP) controls how intelligence can be shared. CLEAR = unrestricted, GREEN = community sharing, AMBER = limited distribution, RED = internal only. Filter searches by TLP to respect data handling agreements.

Q: How often are connectors updated?

A: Connectors run on a schedule (6–24 hours). Admin controls frequency. Check Connector Status dashboard for last-run timestamp.

Q: Can I bulk import IOCs?

A: Yes. In Cases, click "Bulk Import" and paste newline-separated IOCs. SOCint deduplicates and links to your case.

Q: What happens if an IOC is too old?

A: SOCint ages IOCs daily. After 90 days without sighting, confidence drops. Sightings reset the clock (unless case reference exempts).

Q: How do I share intelligence with other organizations?

A: Export cases or IOCs. Filter by TLP:CLEAR or TLP:GREEN for shareable data. Never export TLP:RED externally.

Q: Can I create detection rules without writing Sigma?

A: Admin provides template rules. Clone and edit. Admins can also set up rule libraries from external sources (SIGMA repository).

Q: How do I integrate SOCint with my SIEM?

A: Contact admin to enable connectors (Splunk, ELK, Graylog). Use Sigma rules export. Or build custom integration via REST API.

Q: What's the API rate limit?

A: Contact admin. Default is 1000 requests/minute per user. Bulk operations are queued.

Q: I lost my API key. What do I do?

A: Go to Settings → API Keys → revoke lost key → create new key. Old key is invalidated immediately.

Q: Search returns no results. Why?

A: Check TLP filters (your role may not see all levels). Try broader search term. Ask admin if connectors are running.

Q: Can I export a case as a report?

A: Yes. Click Case → Export → choose PDF or JSON. PDF includes observables, tasks, comments, compliance tags.
